## Project Overview
The purpose of Assessment 3: Responsive website was to continue redesigning and developing the Canberra Modern website. This was done to improve the user experience, make the site more accessible and ensure that it is adaptable to all screen sizes and devices. This assessment was built on the assessments 1 and 2. During development, I followed the high-fidelity Figma prototype created in Assessment 2 to maintain consistency between the planned design and final coded website.


The final website was developed using HTML5 for the structure and CSS3 for the styling and responsive layout design. The website included five pages: Home, About, Events, Multimedia and Contact.


## Design and Development Process
During the design and development process, I aimed to create a modern and clean website inspired by architectural design styles. I chose white and soft green backgrounds throughout the site to create a calm visual cohesive look. To enhance contrast I used pink as the accent colour for the buttons, hover states and interactive elements.


For typography, I chose Playfair Display for headings to give the website an interesting architectural feel. For the body text I used Manrope to improve readability and maintain a clean modern presentation. Overall my goal was to make the website visually appealing and engaging while keeping it simple, fresh and easy for users to navigate across all pages.


## Responsive Design
Responsive design was an important part of the website development process, as the site needed to function effectively across desktop, mobile and tablet devices. To achieve this, I used media queries throughout the CSS stylesheet to adjust layouts, typography and spacing for different screen sizes. Flexbox and CSS Grid were also used to create flexible layouts that could adapt depending on the device being used.


Responsive features were used throughout the website, including a mobile burger navigation menu, responsive event cards and adaptable multimedia layouts. On smaller screen sizes, content such as cards and footer sections stacked vertically to improve usability and readability. Hero text and spacing were also adjusted across devices to maintain consistency and accessibility. Throughout development, the website was regularly tested on both desktop and mobile layouts to ensure the responsive design functioned correctly.


## Challenges and Problem Solving
One challenge throughout the project was ensuring the website remained visually consistent and responsive across all devices. During development, I experienced issues with inconsistent hero section layouts, event card sizing on mobile devices and image scaling. Some images would be cropped differently depending on the screen width, causing main parts of the image to be cut off on smaller devices. An example was the Parliament House images used in the about page sections where the images were hidden or awkwardly cropped across different layouts. To solve this, I adjusted the image positioning and sizing using CSS properties like object-position, object-fit: cover, responsive widths and media queries to ensure the images scaled correctly across mobile and desktop layouts.


Another challenge was managing CSS conflicts and repeated styling rules while developing multiple pages. As more pages and responsive elements were added, some sections began to override each other partially within the navigation layouts and hero sections. To be organised and consistent, I created shared classes and reusable styling elements such as buttons, hero sections, navigation menus and cards.


Developing the mobile burger menu was also challenging; the menu was not displaying correctly across all pages. Throughout troubleshooting and testing, I was able to refine the HTML structure and responsive CSS rules to create a functioning mobile navigation.
